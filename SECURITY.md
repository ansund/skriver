# Security Policy

If you discover a security issue, please do not open a public issue first.

Instead, contact the maintainer privately with:

- a description of the issue
- severity and affected versions
- reproduction steps if possible
- any suggested mitigation

Until a dedicated private contact address is published, please use GitHub security advisories if available for the repository, or contact the maintainer directly.

## Scope

Relevant issues include:

- credential leaks
- unsafe handling of local files
- path traversal or command injection risks
- unwanted network calls
- security-sensitive install behavior
